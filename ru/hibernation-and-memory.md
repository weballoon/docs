# Спящий режим и память

weballoon ограничивает число полностью активных встроенных сессий, чтобы компьютер оставался отзывчивым.

## Как это работает

- При **открытом приложении** на переднем плане часть других сессий остаётся «живой» для быстрого переключения; остальные могут **уходить в спящий режим**.
- На экране **Мои приложения** (ни одно приложение не на весь экран) активных фоновых сессий ещё меньше.
- **Спящий режим** — сессия приостановлена: weballoon запоминает место и может **отключить звук** до следующего открытия. При повторном открытии возможна короткая перезагрузка.

Приложения с **Работа в фоне** в **Настройках → Уведомления** обрабатываются особо: остаются загруженными для уведомлений. Память всё равно используется — см. подсказку в настройках (**50–150 МБ** на фоновое приложение).

## Что делать

- При нехватке ресурсов уменьшите число приложений с **Работа в фоне**.
- Закрывайте ненужные виды приложений (**Закрыть приложение** в заголовке).
- Используйте **Диспетчер задач** в Настройках.
