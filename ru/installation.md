# Установка и поддерживаемые платформы

Официальные сборки для **macOS**, **Windows** и **Linux**, архитектуры **x64** и **ARM64** (Apple Silicon), где применимо.

## macOS

Установщики в форматах **DMG** и **ZIP** (обе архитектуры в релизных сборках). После открытия DMG перетащите **weballoon** в Программы (или по своему обычаю).

**Deep link:** приложение регистрирует схему `weballoon://`, чтобы **magic link** из почты открывал приложение.

## Windows

Формат релиза — **NSIS** (установщик) для x64 и arm64.

Та же схема **`weballoon://`** для завершения привязки по magic link.

## Linux

Релизы в формате **AppImage** (x64 и arm64). Автообновления внутри приложения на Linux могут работать только при установке из поддерживаемого релиз-пакета (приложение сообщит, если обновления недоступны).

## Обновления

В **Настройках**, в области аккаунта, карточка **Обновления приложения** позволяет **проверить обновления** и при готовой загрузке **перезапустить для обновления**. Фоновая загрузка может вызвать предложение перезапуска, когда вам удобно.

В некоторых средах (например локальная разработка) автообновления недоступны; карточка объяснит причину.

## Системные требования

weballoon — десктопный веб-рантайм. Для комфортной работы достаточно обычного современного ПК с несколькими ГБ свободной RAM; каждое открытое или фоновое приложение добавляет нагрузку на память (см. [Спящий режим и память](hibernation-and-memory.md)).
