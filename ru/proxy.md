# Настройка прокси

Откройте **Настройки → Прокси**.

## Маршрутизация по приложению

Каждое приложение может идти через **свой** прокси или **без прокси**. Полезно для:

- Корпоративного HTTP-прокси только для рабочих приложений.
- SOCKS для отдельных инструментов.
- Региональных проверок вместе с изолированными сессиями.

## Поля (прокси включён)

- **Протокол** — HTTP, HTTPS, SOCKS4 или SOCKS5.
- **Хост** и **Порт**.
- **Имя пользователя** (необязательно).
- **Пароль** (необязательно). Интерфейс поясняет: оставить пустым — сохранить прежний пароль; есть действие **очистить сохранённый пароль**.
- **Правила обхода** — список через запятую (пример в приложении: `localhost, 127.0.0.1, *.internal`).

## Сохранение

**Сохранить и перезагрузить** сохраняет настройки и **перезагружает это приложение**, чтобы новые соединения шли через прокси. Предупреждение: изменения **разрывают текущие соединения** затронутого приложения.

## Безопасность

В разделе **Аккаунт** указано: пароли прокси **шифруются на устройстве** и **не** синхронизируются с облаком.

## Пустой список

Сначала добавьте хотя бы одно приложение — так написано на экране прокси.
