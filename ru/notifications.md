# Уведомления

weballoon разделяет два понятия: **системные уведомления** сайтов с Web Notifications API и **список уведомлений в приложении**, чтобы не искать всплывающие сообщения.

## Центр уведомлений в приложении

- Нажмите **колокольчик** в доке или **Shift+N** (если фокус позволяет; см. [Сочетания клавиш](keyboard-shortcuts.md)).
- Можно **отметить всё прочитанным**, открыть запись для перехода к приложению или убрать записи.
- Пока вы **уже смотрите** приложение, новые уведомления от него **не дублируются** во внутреннем списке (системное уведомление сайт может всё равно показать).

Хранится до **100** последних записей.

## «Работа в фоне» (на приложение)

В **Настройках → Уведомления** у каждого приложения переключатель с английской подписью **Run in background**. Если включён:

- Сессия может оставаться активной для **websockets** и т. п., пока вы в другом приложении или пространстве.
- weballoon напоминает: каждое фоновое приложение — примерно **50–150 МБ** памяти (оценка; зависит от сайта).

Без приложений панель просит сначала добавить приложения.

## Практика

- Включайте **Работа в фоне** для чата или почты, если нужны оповещения.
- Отключайте для тяжёлых или редких сайтов, чтобы экономить RAM.

См. также [Спящий режим и память](hibernation-and-memory.md).
