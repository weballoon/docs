# Raccourcis clavier

Les raccourcis ci-dessous correspondent à l’implémentation **actuelle** de weballoon. Ils supposent que vous ne **tapez pas** dans un champ de texte, une recherche ou autre contrôle éditable — sinon, les lettres peuvent aller au site.

## Changement d’espace de travail

| Action | Raccourci |
|--------|-----------|
| **Espace suivant** (cycle) | **Maj+W** |
| **Aller aux espaces 1–9** | **Ctrl+1** … **Ctrl+9** sous Windows/Linux, **⌘1** … **⌘9** sur macOS — **uniquement lorsque le focus est dans une app web ouverte** (pour éviter les conflits avec les écrans propres à weballoon). |

Après un cycle avec **Maj+W**, tant que vous maintenez **Maj**, une petite **superposition** de sélection d’espace peut apparaître jusqu’à relâchement de **Maj**.

Depuis **Mes apps**, **Catalogue** ou **Réglages**, utilisez le bouton **espace de travail** ou **Maj+W** si les raccourcis numériques ne réagissent pas.

| Action | Raccourci |
|--------|-----------|
| **Afficher ou masquer le centre de notifications** | **Maj+N** |

## Image dans l’image (panneau vidéo)

| Action | Raccourci |
|--------|-----------|
| **Afficher ou masquer le flottant PiP** | **Maj+S** |

## Dock

- L’info-bulle du bouton **Espace de travail** indique **Maj+W**.
- Celle de la cloche **Notifications** indique **Maj+N**.

## Barre de titre

Les boutons de navigation (**Précédent**, **Accueil**, **Suivant**, **Actualiser**) sont **souris / pointeur uniquement** dans l’UI actuelle — pas de rangée clavier documentée à part.

Si un raccourci ne fait rien, cliquez d’abord sur le fond de weballoon ou la barre de titre pour que le focus ne soit pas dans un champ du site.
