# Protection par code PIN

Ouvrez **Réglages → Protection**.

## Rôle

Vous pouvez protéger des apps sensibles avec un **code PIN numérique à 6 chiffres**.

- Activez **Activer la protection**. S’il n’y a pas encore de PIN, weballoon vous guide pour en **définir** un ; s’il existe déjà, vous devez **vérifier** avant d’activer ou désactiver la protection.
- Utilisez **Configurer le PIN** ou **Changer le PIN** pour gérer le code (avec étapes de confirmation).

## Verrous par app

Plus bas, sous **Apps protégées**, activez l’interrupteur pour chaque app devant exiger le PIN **à l’ouverture**.

Les interrupteurs par app sont **désactivés** tant que la protection globale n’est pas activée (la section est estompée jusqu’à activation).

## Quand le PIN apparaît

Ouvrir une app **protégée** affiche un écran d’**authentification** jusqu’à saisie du bon PIN. Après un déverrouillage, l’app reste déverrouillée **pour le reste de la session** jusqu’à redémarrage de weballoon ou équivalent (pas besoin de ressaisir le PIN à chaque retour sur l’app dans la même exécution).

## Si vous oubliez le PIN

L’app n’expose pas de « clé maître » dans l’interface. Si vous êtes bloqué, il vous faudra la voie de récupération fournie par le produit (support officiel ou procédure documentée). **Évitez** les essais répétés au hasard ; conservez une copie de secours du PIN en lieu sûr.
