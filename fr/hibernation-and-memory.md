# Hibernation et mémoire de l’app

weballoon limite le nombre de sessions intégrées **pleinement actives** à la fois pour que l’ordinateur reste réactif.

## Comportement (langage simple)

- Avec **une app ouverte** devant vous, weballoon garde un petit nombre d’autres sessions « vivantes » pour un changement rapide ; les suivantes peuvent **hiberner**.
- De retour sur **Mes apps** (aucune app en plein écran), encore moins de sessions non essentielles restent vivantes en arrière-plan.
- **Hiberner** ici signifie mettre la session en pause : weballoon enregistre où vous étiez et peut **couper le son** de cette session jusqu’à la réouverture. Rouvrir peut provoquer un bref rechargement.

Les apps avec **Exécuter en arrière-plan** activé dans **Réglages → Notifications** sont traitées à part : elles restent montées pour le comportement lié aux notifications. Elles comptent toujours pour la mémoire — voir la note dans Réglages (environ **50–150 Mo** par app en arrière-plan).

## Que faire

- Si la machine est lourde, réduisez le nombre d’apps **Exécuter en arrière-plan**.
- Fermez les vues d’app inutiles (**Fermer l’app** dans la barre de titre).
- Utilisez le **Gestionnaire des tâches** dans Réglages pour voir quelles apps consomment le plus de CPU ou de RAM.
